# Go Conventions — derived from addit-digital/app-erp

Extracted from the live codebase. This is the global baseline; a per-project
`.claude/go-conventions.md` (generated by `/go-conventions`) overrides or extends
these for a specific repo.

> **Stack:** Go 1.24 · Gin · MongoDB (mongo-driver/v2) · `google/uuid` ·
> `greatcloak/decimal` · `log/slog` + OTel · `go-playground/validator/v10`

---

## 1. Project layout

```
cmd/          → main package (entry point, wiring only)
internal/     → shared infrastructure not exported outside the module
  database/   → MongoDB client + generic cursor/record helpers
  errors/     → custom error type + YAML-driven error registry
  event/      → in-process event bus (publisher + subscriber)
  i18n/       → translation bundle
  logger/     → slog-based logger with OTel bridge
  property/   → YAML config loader
  validator/  → request validation bootstrap
pkg/
  {domain}/   → one package per business domain (sales, invoice, product …)
  clients/    → external HTTP/API clients (Cloudflare, Brevo, R2)
  events/     → event struct definitions (one file per domain)
  handlers/   → Gin HTTP handlers + router wiring
  iam/        → auth context helpers, JWT, session
  middlewares/→ Gin middleware (auth, CORS, error, feature gates)
  models/     → shared API DTOs (request/response types with json tags)
  dtos/       → filter/query request DTOs
  validators/ → custom validator.Func implementations
```

## 2. Domain package structure

Every domain package (`pkg/{domain}/`) contains exactly these files:

| File | Contents |
|------|----------|
| `model.go` | MongoDB entity structs (`bson` tags) + internal types (`UpdateXxx`, `FilterXxx`) |
| `service.go` | `Service` interface + private `service` impl + `Builder` struct + `NewService` |
| `repository.go` | `Repository` interface + private `repository` impl + `NewRepository` |
| `mapper.go` | `ToXxxResponse`, `FromXxxRequest` pure mapping functions |
| `listener.go` | `EventListener` impl + `XxxListenerBuilder` + `NewXxxListener` |
| `lifecycle.go` | Lifecycle resolver (optional — only for state-machine entities) |

Do not add files outside this set without a clear reason.

## 3. Naming

- **Packages:** lowercase singular — `sales`, `invoice`, `product` (not `invoices`)
- **Files:** snake_case — `model.go`, `sales_order.go`, `error_handler.go`
- **Types:** PascalCase — `SalesOrder`, `UpdateSalesOrder`, `FilterDiscount`
- **Constants:** PascalCase for typed constants — `SalesOrderStatusDraft`;
  SCREAMING_SNAKE_CASE only for untyped string literals used as codes or keys
- **Interfaces:** noun or noun-verb — `Service`, `Repository`, `EventPublisher`
- **Handler functions:** `handle{Action}{Resource}` — `handleCreateDraftSalesOrder`,
  `handleGetSalesOrderList`
- **IDs:** always `uuid.UUID` (`github.com/google/uuid`), never bare `string`
- **Decimals:** always `decimal.Decimal` (`github.com/greatcloak/decimal`), never
  `float64` for money or quantities

## 4. Dependency injection — Builder pattern

Every service and repository uses an embedded `Builder` struct for DI:

```go
type Builder struct {
    SalesRepository Repository
    EventPublisher  event.EventPublisher
    CounterService  counter.Service
    CustomerService customer.Service
}

type service struct {
    Builder
}

func NewService(builder Builder) Service {
    return &service{Builder: builder}
}
```

- The `Builder` struct is exported; the `service` / `repository` impl is unexported.
- All dependencies are interfaces, never concrete types.
- Wire everything in `cmd/app.go`; never init dependencies inside a service.
- Optional deps (e.g. external clients) are passed as pointers — check for nil before use.

## 5. Model / DTO separation

**Rule: never expose database entities in API responses.**

| Layer | Location | Tags |
|-------|----------|------|
| MongoDB entity | `pkg/{domain}/model.go` | `bson` |
| Shared embedded types (multi-domain) | `pkg/models/` | `bson` + `json` |
| API request/response DTOs | `pkg/models/` | `json` |
| External client request/response | `pkg/models/{client}.go` | varies |
| Filter/query DTOs | `pkg/dtos/` | `json` |

Mappers live in `pkg/{domain}/mapper.go` — named `ToXxxResponse` and
`FromXxxRequest`. They are pure functions, no side effects.

## 6. Error handling

### Custom error type

```go
// internal/errors/model.go
type Error struct {
    Id         string         `json:"id"`
    Code       string         `yaml:"code" json:"code"`
    Message    string         `json:"message"`
    StatusCode int            `yaml:"status_code" json:"status_code"`
    Params     map[string]any `json:"params,omitempty"`
}

func (e Error) Error() string { return e.Code }
```

### Error definitions

Errors are defined in `error.yaml` and loaded at startup with `errors.InitErrors()`.
Sentinel errors in `pkg/models/`:

```go
var ErrPermissionDenied = errors.New("PERMISSION_DENIED")
var ErrSalesOrderNotFound = errors.New("SALES_ORDER_NOT_FOUND")
```

### Service error pattern

```go
func (s *service) DoThing(ctx context.Context, id uuid.UUID) (*models.ThingResponse, error) {
    log := logger.GetLogger(ctx)
    thing, err := s.Repository.FindById(ctx, id)
    if err != nil {
        log.Errorf("Failed to find thing: %v", err)
        return nil, err
    }
    return ToThingResponse(thing), nil
}
```

- Always log before returning an error: `log.Errorf("Failed to X: %v", err)`
- Never wrap errors (`fmt.Errorf("...: %w", err)`) — propagate the typed `Error` as-is
- Enrich with params before returning: `models.ErrInvalidAccountType.AddParam("AccountType", ...)`
- Use `errors.Is(err, mongo.ErrNoDocuments)` for not-found detection in repos

### Handler error pattern

```go
func (h *handler) handleCreateThing(c *gin.Context) {
    var request models.CreateThingRequest
    if err := c.ShouldBindJSON(&request); err != nil {
        _ = c.Error(err)
        return
    }
    err := validator.ValidateRequest(request)
    if err != nil {
        _ = c.Error(err)
        return
    }
    result, err := h.ThingService.Create(c.Request.Context(), request)
    if err != nil {
        _ = c.Error(err)
        return
    }
    c.JSON(201, result)
}
```

- Handlers **never** write status codes for errors — always `_ = c.Error(err); return`
- The `ErrorHandler` middleware dispatches by type and writes the JSON response
- Parse UUID params: `c.Param("id")` → `uuid.Parse(...)` → `c.Error(models.ErrInvalidParameter)` on failure

## 7. Logging

```go
log := logger.GetLogger(ctx)   // at the top of every method that logs

log.Infof("Creating sales order for customer %s", customerId)
log.Warnf("Failed to get business config, using default: %v", err)
log.Errorf("Failed to save sales order: %v", err)
```

- Always acquire the logger from context: `logger.GetLogger(ctx)` — it carries correlation ID + user info
- Use `f`-variants for format strings; use plain variants for structured: `log.Info("msg", slog.String(k, v))`
- Local env: `slog.NewJSONHandler(os.Stdout)`; prod: OTel OTLP bridge
- Do **not** use `logrus` in new code — it is a legacy import, slog is the standard

## 8. MongoDB patterns

```go
const collectionSalesOrder = "sales_orders"   // package-level constant

// Build filter incrementally
filterBson := bson.M{
    "active":      true,
    "business_id": businessId,
}
if filter.Status != nil {
    filterBson["status"] = *filter.Status
}

// FindOneAndUpdate — always return the updated document
result := r.database.Collection(collectionSalesOrder).FindOneAndUpdate(
    ctx,
    bson.M{"_id": id, "active": true},
    bson.M{"$set": updateQuery},
    options.FindOneAndUpdate().SetReturnDocument(options.After),
)

// Cursor helper
items, err := database.DecodeCursor[*SalesOrder](ctx, cursor)

// Pagination
cursor, err := collection.Find(ctx, filter,
    options.Find().
        SetSort(bson.M{"updated_at": -1}).
        SetLimit(int64(pageRequest.Size)).
        SetSkip(int64(pageRequest.Page * pageRequest.Size)),
)

// Not-found detection
if errors.Is(err, mongo.ErrNoDocuments) {
    return nil, nil
}
```

- Use `go.mongodb.org/mongo-driver/v2` (the v2 driver), not the legacy v1
- Soft deletes only: `active: false`, never hard-delete documents
- Domain entities implement `database.Model` (`GetId()`/`SetId()`) for the generic helpers
- Field names in BSON: `snake_case` matching the struct tags
- Pagination result: `models.Page[T]` with `HasNext`, `Total`, `TotalPages`, `CurrentPage`, `Size`, `Data`

## 9. HTTP / Gin patterns

```go
// Router group pattern
salesRouter := router.Group("/api/v1/sales")
salesRouter.Use(middlewares.ValidateToken(authService))
salesRouter.Use(middlewares.RequireFeature(models.ModuleSales, planService))
salesRouter.Use(middlewares.RequireAccess(models.ModuleSales, models.AccessReadOwn))
salesRouter.POST("/orders", middlewares.RequireAccess(models.ModuleSales, models.AccessWrite), h.handleCreateDraftSalesOrder)
```

- All routes under `/api/v1/{resource}` prefix
- Auth middleware applied at group level; per-route access tightened via additional middleware
- HTTP methods: `POST` for create + list (list uses body filter), `GET` for single fetch,
  `PUT` for full update, `PATCH` for partial update, `DELETE` for deactivate
- Response codes: `201` for create, `200` for everything else
- List endpoints use `POST /list` with a body filter (not query params)

## 10. Event system

```go
// Define event in pkg/events/{domain}.go
type SalesOrderConfirmed struct {
    SalesOrderId uuid.UUID
    BusinessId   uuid.UUID
}
func (e *SalesOrderConfirmed) EventName() string { return "SalesOrderConfirmed" }
func (e *SalesOrderConfirmed) IsAsync() bool      { return false }

// Publish after success — errors intentionally discarded
_ = s.EventPublisher.Publish(ctx, &events.SalesOrderConfirmed{
    SalesOrderId: order.Id,
    BusinessId:   userContext.BusinessId,
})

// Listener in pkg/{domain}/listener.go
type Listener struct { ListenerBuilder }
type ListenerBuilder struct { SomeService SomeService }
func NewListener(b ListenerBuilder) *Listener { return &Listener{b} }
func (l *Listener) Listen(ctx context.Context, event event.Event) error { ... }
```

- Sync events (`IsAsync() false`): blocking, errors propagated, use for side-effects that must succeed
- Async events (`IsAsync() true`): fire-and-forget goroutine with `context.WithoutCancel(ctx)`
- Subscribe in `cmd/app.go` via `eventSubscriber.Subscribe(ctx, listener, events.XxxEvent, ...)`

## 11. Context & IAM

```go
userContext := iam.GetUserContext(ctx)
if !userContext.Role.AtLeast(models.UserRoleSales) {
    return nil, models.ErrPermissionDenied
}
// Access business-scoped data
repo.Find(ctx, id, userContext.BusinessId)
```

- `ctx` is always the first parameter of every service and repository method
- User context is extracted from context, never passed as a plain parameter
- `userContext.BusinessId` always scopes DB queries — never trust client-provided businessId for data access
- Module-level access: `iam.GetModuleAccess(ctx)` returns `AccessRead`, `AccessWrite`, `AccessAdmin`, `AccessReadOwn`

## 12. Validation

```go
// Request structs use go-playground/validator tags
type CreateThingRequest struct {
    Name string         `json:"name" validate:"required,min=2,max=200"`
    Amount models.Money `json:"amount" validate:"required"`
}

// Custom validators registered in cmd/app.go
validate.RegisterValidation("minDecimal", validators.MinDecimalValidator)

// Handler: always validate after binding
if err := c.ShouldBindJSON(&request); err != nil { ... }
if err := validator.ValidateRequest(request); err != nil { ... }
```

## 13. Import order

```go
import (
    // 1. stdlib
    "context"
    "errors"
    "time"

    // 2. internal packages
    "github.com/addit-digital/app-erp/internal/event"
    "github.com/addit-digital/app-erp/internal/logger"

    // 3. pkg packages
    "github.com/addit-digital/app-erp/pkg/models"

    // 4. third-party
    "github.com/google/uuid"
    "go.mongodb.org/mongo-driver/v2/bson"
)
```

Use alias imports only to resolve naming collisions: `tmpl "github.com/addit-digital/app-erp/pkg/template"`.

## 14. Key third-party library patterns

| Library | Pattern |
|---------|---------|
| `google/uuid` | `uuid.New()` for generation; `uuid.Parse(s)` returns `(uuid.UUID, error)` |
| `greatcloak/decimal` | `decimal.Zero`, `decimal.NewFromInt(n)`, `.Mul/.Add/.Sub/.Round(precision)` |
| `gin-gonic/gin` | `c.ShouldBindJSON`, `c.JSON`, `c.Param`, `c.Error`, `c.Request.Context()` |
| `mongo-driver/v2` | `bson.M` for filters; `options.Find().SetSort/SetLimit/SetSkip` for queries |
| `go-playground/validator/v10` | struct tags; custom funcs registered at startup |
| `honeycombio/otel-config-go` | single-call OTel setup; shutdown deferred in `main` |
